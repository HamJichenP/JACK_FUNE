# Discord 機器人與 Google 試算表 (GAS) 串接說明文件

本文件旨在說明 Discord 機器人如何將成員的報名資料傳送至 Google 試算表，以及傳送的參數格式，以便試算表設計者進行公式運算、排版與統計。

---

## 📡 資料傳輸規格

* **傳輸協議**：HTTP POST 請求
* **資料格式 (Content-Type)**：`application/json` (UTF-8 編碼)
* **API 接收端**：試算表後台部署的「Google Apps Script (GAS) 網頁應用程式網址」

---

## 📥 機器人傳送的資料參數 (JSON Payload)

機器人每次呼叫 API 時，會傳送一筆 JSON 格式的資料，包含以下三個參數：

| 參數名稱 (Key) | 資料型別 | 說明與範例 |
| :--- | :--- | :--- |
| `name` | 字串 (String) | 成員在 Discord 伺服器中的**「顯示名稱 (暱稱)」**。<br>例如：`"戰神小明"`。 |
| `roles` | 字串 (String) | 成員所擁有的身分組。<br>1. 若管理員有設定「篩選身分組白名單」，則只會包含白名單內的身分組。<br>2. 多個身分組會以**「半形逗號加空白」**串接，例如：`"VIP成員, 戰報組"`。<br>3. 若無符合的身分組，則固定傳送 `"無身分組"`。 |
| `day` | 字串 (String) | 成員點選的報名類別。<br>* 點選星期六按鈕：傳送 `"星期六"`<br>* 點選星期日按鈕：傳送 `"星期日"`<br>* 點選表情符號：傳送 `"反應報名"` |

### 範例 JSON 封包：
```json
{
  "name": "戰神小明",
  "roles": "VIP成員, 戰報組",
  "day": "星期六"
}
```

---

## ⚙️ Google Apps Script (GAS) 的資料寫入邏輯

當試算表後台的 GAS 接收到上述 JSON 封包時，會依序執行以下動作：

1. **啟用「排隊防衝突鎖 (LockService)」**：
   因為可能有多個成員同時點擊按鈕，腳本會強制寫入動作排隊執行（最長等待 30 秒），保證每一筆資料都會被安全寫入，**絕不覆蓋**。
   
2. **根據日期 (day) 進行「局部向下推擠 (insertCells)」**：
   * **當為「星期六」時**：
     * 僅針對 **A2:B2** 以下的儲存格進行向下推擠，保留 C 欄與 D-E 欄不被往下推。
     * 將 `name` (遊戲ID) 寫入 **A2**，`roles` (武學) 寫入 **B2**。
   * **當為「星期日」或「反應報名」時**：
     * 僅針對 **D2:E2** 以下的儲存格進行向下推擠，保留 A-B 欄與 C 欄不被往下推。
     * 將 `name` (遊戲ID) 寫入 **D2**，`roles` (武學) 寫入 **E2**。

---

## 📊 試算表設計師排版建議

* **首欄設定**：
  * **A1**：`【週六待編制區】` / **A2**：`遊戲ID` / **B2**：`武學`
  * **D1**：`【週日待編制區】` / **D2**：`遊戲ID` / **E2**：`武學`
* **統計與公式**：
  因為新資料會以「局部在第二列 (A2:B2 / D2:E2) 插入並向下推擠」的方式寫入，因此如果您有使用統計公式（如 `COUNTA`、`COUNTIF`），**建議將公式放置在其他工作表分頁**，或公式直接對**整欄**進行運算（例如：`=COUNTIF(A:A, "<>")`），避免公式範圍因儲存格推擠而移位。
